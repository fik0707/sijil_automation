1. Create Google Form

Setup

Configure Your Google Form

  1. Go to Google Forms and create a new form.
  2. Add a question named "Nama Penuh" (or Full Name).
  3. Set the question type to Short Answer.
  4. Toggle the Required (Wajib) switch to active so participants must fill in their name.

💡 Pro-Tip

Keep the name placeholder consistent. By default, our script will search for the name column in the spreadsheet generated by this form.

2. Collect Email Addresses

Setup

Enable Email Collection

To send certificates, we must have the participant's email address. Google Forms handles this automatically.

  1. At the top of your Google Form, click on Settings (Tetapan).
  2. Expand the Responses (Jawapan) section.
  3. Find the option Collect email addresses and choose either:
    • Verified (Requires Google Account login - recommended for organization security)
    • Responder input (Adds an email question field manually - best for public forms)
  • ✓
    Email Collection Enabled Set to either "Verified" or "Responder input" in Form Settings.

3. Create Output Certificate Folder

Drive Configuration

Prepare Certificate Storage

All generated certificate PDFs will be stored in this Google Drive folder. Setting this up keeps your files organized.

  1. Go to Google Drive.
  2. Click New > New folder and name it (e.g., Sijil Peserta Bengkel).
  3. Open the folder, and copy the full URL from your browser's address bar.

💡 Security Tip

Make sure this folder is secure. You do not need to share this folder with participants; the script will email them the PDFs directly.

4. Create Slides Certificate Template

Design Template

Design Your Certificate

We use GOOGLE SLIDES as our certificate template. The script will duplicate this template and replace your text placeholder with the participant's name.

  1. Create a new Google Slide presentation (if using PowerPoint, click File (Fail) and select Save as Google Slides (Simpan sebagai Google Slaid)).
  2. Design your layout and insert a textbox containing your name placeholder, for example: <<FULL NAME>>.
  3. Click the Share button (top right), change General Access to "Anyone with the link", and copy the link.

5. Link Form to Google Sheet

Data Link

Retrieve Submissions Sheet

Form submissions must flow into a spreadsheet so the Apps Script can read and process them.

  1. In your Google Form, click the Responses (Jawapan) tab.
  2. Click the green Link to Sheets (Hubungkan ke Helaian) button.
  3. Choose Create a new spreadsheet and click Create.
  4. Note the columns in the generated Sheet:
    • Typically: Column A is Timestamp
    • Column B is Email Address (Index 2)
    • Column C is Nama Penuh (Index 3)

💡 Mapping Help

Column numbers start at 1. Column A = 1, B = 2, C = 3, D = 4, etc. Double check which columns contain the Email and the Name.

How to Set Up the Script (Easy Step-by-Step for Beginners):

  1. Open the Google Sheet that is linked to your Form (from Step 5).
  2. At the top menu bar, click on Extensions (Sambungan), then click on Apps Script.
  3. A new web page will open. In the center, you will see a default code editor containing: 
    function myFunction() {

}
  4. Select all of that code (press Ctrl + A on Windows or Cmd + A on Mac) and delete it completely, so the editor is completely empty.
  5. Customize your details in the configuration panel above, click the Copy Code button, and paste it (press Ctrl + V or Cmd + V) into the empty editor.
  6. Save the script (click the Save floppy disk icon at the top menu).

7. Set Up Automation Trigger

Triggers

Create Submit Trigger

This tells Google to automatically execute our script whenever a participant submits a new Google Form response.

  1. Inside the Apps Script editor, click on the **Triggers (Pencetus)** icon in the left sidebar (the alarm clock icon ⏰).
  2. Click the blue + Add Trigger (+ Tambah Pencetus) button in the bottom right.
  3. Configure the trigger settings exactly as follows:
    • Choose which function to run: sendCertificate
    • Choose which deployment should run: Head
    • Select event source: From spreadsheet (Dari helaian)
    • Select event type: On form submit (Semasa menghantar borang)
  4. Click Save.
  • ✓
    Alarm Clock Icon Clicked Opened the Triggers dashboard in the script editor.
  • ✓

8. Save & Authorize Script

Authorization

Initial Authorization Run

Google requires you to manually approve the permissions for the script to access your Sheets, Drive, Slides, and send emails.

  1. Go back to the Apps Script Editor (</>) view.
  2. Click the Save (Disimpan) floppy disk icon at the top menu.
  3. Select sendCertificate in the dropdown menu next to the "Debug" button, then click Run (Jalan).
  4. A modal popup will say "Authorization Required". Click Review Permissions.
  5. Choose your Google Account.
  6. On the warning screen ("Google hasn't verified this app"), click Advanced at the bottom left, then click Go to Untitled project (unsafe).
  7. Click Allow on the final permissions screen.
  • ✓
    Script Saved Saved the code changes in the editor.
  • ✓
    Run Clicked & Authorized Allowed access permissions for Drive, Slides, and Mail services.

⚠️ Important Note

The execution log will show an error Ralat: Tiada data... if your Sheet is completely empty during the test run. This is normal! As long as the authorization succeeded, you are ready.
